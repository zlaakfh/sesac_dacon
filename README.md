# 🚗 **See:Drive — 딥러닝 기반 도로 환경 사고 판단 및 자동 신고 시스템**

Vision AI + IMU 센서 융합 기반 **자율형 사고 인지·판단·대응 통합 서비스**

---

## **1. 프로젝트 개요**

**See:Drive**는
Vision AI와 차량 IMU 센서를 융합하여 **도로 사고를 자동으로 탐지·판단·신고**하며,
추가적으로 **후진 주차 중 발생하는 접촉 사고까지 예방**할 수 있는 **지능형 차량 안전 서비스**입니다.

본 서비스는

* 사고 발생 직후 **골든 타임 내 자동 신고**,
* 위험 요소 사전 탐지,
* 주차 중 접촉 사고 예방
  을 목표로 설계되었습니다.

## **3. 핵심 기능 요약**

### **1) 사고 자동 판단·신고 시스템 (주행 사고 중심)**

* Vision 기반 위험 요소 탐지

  * YOLO: 포트홀·도로 파손·장애물 등 즉시 대응 필요한 요소 탐지
  * Detectron2: 연기·불꽃 등 비정형 패턴 정밀 분할

* IMU 센서 기반 사고 여부 판단

  * 가속도·각속도 변화량 분석 → 사고 Level 1~3 분류

* 자동 신고

  * Level 1: 팝업
  * Level 2: 미응답 → GPS 자동 신고
  * Level 3: 즉시 신고 + 음성 탈출 안내

---

### **2) 주차 중 접촉사고 예방 시스템 (사고 예방 기능)**

> *주차 보조 기능이 아니라, ‘후진 접촉사고를 예방하기 위한 사고 예방 모듈’로 정의*

* Segmentation 기반 구조물 인식
* 사용자 선택 기반 ROI 영역 분석
* 중심선·각도 기반 진입 가능성 계산
* AR 기반 후진 경로 제공(곡선 가이드 + 핸들 방향 안내)

---

## **4. 제안 배경**

### ● 반복적으로 발생하는 문제

* 사고 상황에서 운전자가 패닉 상태 → **신고 지연**
* 정확한 위치 전달 불가 → **구조 지연**
* 고령자/초보자는 더욱 취약

### ● 기술 부재 문제

* Vision 분석 + 센서 이상값을 **통합적으로 처리하는 사고 판단 시스템 부재**

### ● 주차 접촉사고 문제

* 전체 사고 중 **30% 이상이 주차 중 발생**
* 기존 주차 보조는 “경고음 + 단순 후방 카메라” 수준
  → **정확한 주차선/각도·거리 기반 안내 기능 없음**

### → 따라서,

본 서비스는 ‘**사고 대응 + 사고 예방(후진 접촉사고)**’까지 포함하는
**AI 기반 통합 안전 솔루션**을 목표로 한다.

## **5. 세부 기능 설계**

---

# **5.1 Vision 기반 사고 전조요인 탐지**

Vision AI가 사고 가능성이 있는 위험 요소를 실시간 분석해
**사고 판단 알고리즘의 1차 입력 신호**로 제공한다.

### [표 1] Vision 입력 → 출력 구조

| 모델         | 탐지 대상         | 출력                | 활용 목적        |
| ---------- | ------------- | ----------------- | ------------ |
| YOLO11n    | 포트홀·파손·차량·장애물 | Bounding Box      | 실시간 위험요소 탐지  |
| Detectron2 | 연기·불꽃·주차선·주차면 | Segmentation Mask | 비정형 패턴 정밀 분할 |

---

# **5.2 센서융합 기반 사고 판단 알고리즘**

Vision에서 위험요소가 탐지되면
IMU 센서(Accel/Gyro) 변화량을 분석하여 사고 여부를 판정한다.

### [표 2] 센서 입력 구성

| 센서         | 축     | 의미          |
| ---------- | ----- | ----------- |
| 가속도(Accel) | X/Y/Z | 전후·좌우·상하 충격 |
| 각속도(Gyro)  | X/Y/Z | 차량 자세 변화    |
| GPS        | 위도·경도 | 사고 위치       |

### [수도코드] 사고 레벨 판정

```python
ΔAccel = |Accel_t - Accel_(t-1)|
ΔGyro  = |Gyro_t  - Gyro_(t-1)|

# Level 1
if ΔAccel > T1 or ΔGyro > T1:
    level = 1

# Level 2
if ΔAccel > T2 and 사용자_미응답:
    level = 2

# Level 3
if ΔAccel > T_high or 연기_탐지 or Z축기울기 > 80°:
    level = 3
```

---

# **5.3 사고 유형 통합 정의 (후진 주차 사고 포함)**

### [표 3] Vision + IMU 기반 사고 유형 분류

| 사고 유형      | Vision 신호  | IMU 신호          | 해석    |
| ---------- | ---------- | --------------- | ----- |
| 포트홀 충격     | 요철/파손 인식   | ΔAccel 증가       | 하부 충격 |
| 전방 충돌      | 차량/벽 접근    | ΔAccel X, Z 급상승 | 추돌    |
| 화재         | 연기/불꽃 Seg. | 변화 없음           | 화재 위험 |
| 후진 주차 접촉사고 | 기둥/차량 근접   | ΔAccel 소폭 증가    | 저속 접촉 |

---

# **5.4 사고 대응 단계 (Popup → 자동 신고)**

### [표 4] 사고 대응 단계

| Level   | 조건                        | 동작            |
| ------- | ------------------------- | ------------- |
| Level 0 | Vision 전조 감지              | 센서 분석 시작      |
| Level 1 | ΔAccel > T1               | 사고 여부 확인 팝업   |
| Level 2 | ΔAccel > T2 or ΔGyro > T3 | 자동 신고         |
| Level 3 | 화재·전복 등                   | 즉시 신고 + 음성 안내 |

---

# **5.5 자동 신고 (FastAPI)**

### 전송 데이터

* 사고 이미지 프레임
* GPS(위도/경도)
* ΔAccel·ΔGyro 전체 로그
* 사고 레벨
* 사고 시각

---

# **5.6 후진 주차 접촉사고 예방 모듈 (사고 예방 기능)**

> **사고 시스템의 하위 기능 중 하나**

### [표 5] 주차 접촉사고 예방 구성 요소

| 요소       | 입력값               | 역할             |
| -------- | ----------------- | -------------- |
| 구조물 인식   | Segmentation Mask | 주차면·주차선·장애물 구분 |
| 빈 공간 탐지  | Polygon           | 주차 가능 영역 분석    |
| 진입 분석    | 중심선·각도·거리         | 진입 가능성 판단      |
| AR 경로    | 거리·각도             | 후진 주차 경로 생성    |
| 충돌 위험 판단 | Offset            | 접촉 위험 평가       |

### [수도코드]

```python
offset = |차량중심선 - 주차면중심선|
angle  = angle(차량_heading, 주차면_heading)

# 진입 가능 여부
if offset < O1 and angle < A1:
    진입가능 = True

# 경로 생성
AR_curve = Bezier(P0=차량, P1=제어점, P2=목표점)

# 충돌 위험 판단
if offset < 최종임계값:
    주차완료 = True
```

📷 Camera
      ↓

  - YOLO: 위험요소 실시간 탐지

      ↓
[센서 융합 사고 판단 알고리즘]

      ↓
